Ruby Hashes vs. JavaScript Objects

Intermission: Line please!

February 14, 2016

Starting with ruby: Hashes are a form of storing data. Hashes have both literal and constructor notations for creating them. There are also a couple different ways to create a hash key value pair and what those can be but as well but just going to focus on a small bit for this blog entry.
  • Creating a Hash
    • 
  literal_hash = {} # this is an empty hash
  hash = Hash.new(0) #this sets the default value of the keys to 0
  hash["a"] = 45      # This is one way to create key pairs
  hash[42] = "life"   # {42 => "life"}
  hash["b"]           # =>0 Creating a key
  hash["c"]           # => 0
       #hash is {a => 45, 42 => life, b => 0, c => 0}

  # This is another way to create a hash and set the keys and values.
  colors = {"red" => [255,0,0], "green" => [0,255,0], "blue" => [0,0,255]}

Moving on to JavaScript Objects are a form of storing data, functions and more object. Woof the sky is the limit here. JavaScript Objects have both literal and constructor notations just like hashes. There is also a nifty way access them with the '.' learning by example type this in to a file or your terminal in node:
